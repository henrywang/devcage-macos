Communication Style

## Response Style
- Be concise, no trailing summaries of what you just did
- Don't add comments to code unless the logic is non-obvious
- Don't use emojis
- When showing file edits, explain why, not just what

---
Coding Preferences

## Code Style
- Prefer explicit over clever
- No unnecessary abstractions for one-off use
- Don't add error handling for impossible scenarios
- No docstrings unless I ask
- Prefer early returns over nested if-blocks

---
Workflow

## Git
- Always write commit messages in imperative mood ("Add X", not "Added X")
- Use conventional commits: feat/fix/chore/docs
- Do not include Co-Authored-By in commit messages

## Before finishing a task
- Run linter if one exists in the project
- Check for obvious security issues (SQL injection, XSS, etc.)

Languages You Use

## Language Preferences

- Shell: POSIX-compatible unless using bash-specific features
